Changes to Performance

After showing the current prototype of the performance page to potential users and having them complete tasks coinciding with the original user interviews , it was found that the design lacked a measure of the cost per ad, which the user cared about on a daily basis. Consequently a section detaling the current day's performance metrics, particularly that of $/Impression was added to the top of the page.

Moreover, the user feedback also revealed that the user would like to be able to filter by the location in which the ad is served, so this functionality was also added.

































Changes to AB Testing

Peer feedback revealed that the create/cancel AB test functionality was not salient, and so the size was increased. Moreover, a progress bar was added to represent the progress towards completion. Peer feedback also noted that the most important piece of information on the page, the result of the test, was not salient enough, and so it was chagned to a bolded font. The test filter was also moved leftwards, right below the first line of text instructing the user to select a test to view. The leftwards orientation will allow for faster use.

The add or remove AB Tests button was also made larger to maintain consistency with the Performance page





















Changes to Ad Management

The add or remove AB Tests button was made larger to maintain consistency with the Performance and AB Testing pages
