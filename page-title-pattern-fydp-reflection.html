Reflection

Given that this project was taken to the final stage of development, it's appropriate to reflect on the deisgn process involved and the next steps.

User requirements were gathered by carrying out interviews with a range of different potential users. In retrospect, it would have been beneficial to also create various Personas in order to understand the needs of a larger variety of users; and to be better able to empathize with them throughout the design process.

Task analysis was then carried out was done using the results from user interviews. This led to the idea of having 2 seperate pages, one to deal with ad performance, and one for AB testing. Looking back, the task analysis could have been refined to be more granular in order to better flush out the functionalities required.

In keeping with the main takeaways from the user interviews and task analysis a basic navigation prototype was made to allow users to access the performance of their ads, manage the set of active ads, and run AB tests using the top bar. A filter tool implemented across each of the pages will allow users to filter between multiple criteria such as dates, and ad campaigns. This navigation prototype could have been improved by exploring other forms of navigation such as left oriented navigation instead of the top bar.

Several mood boards were created to explore different themes that could be incorporated into the final design. These could have been improved through the addition of different fonts as well as a broader variety of images.

Next, wireframes for each main page were created. The wireframes could have been improved by incorporating the prototypes of the charts that were going to be used, as they are a key part of the user experience. The interaction of information between the graphs is also very important, as we have to make sure users are able to quickly get all of the information they need the most. The icons used in these wireframes could also be replaced by text as they do not add enough value to justify the amount of space they take on the screen.

An initial prototype was made and then revised using feedback from users to create a refined design. The refined design could have been further improved through the use of a usability scale survey.
